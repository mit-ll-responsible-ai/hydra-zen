hydra_run¶

hydra_zen.experimental.hydra_run(config, task_function, overrides=None, config_dir=None, config_name='hydra_run', job_name='hydra_run', with_log_configuration=True)[source]¶

(Deprecated) Launch a Hydra job defined by task_function using the configuration provided in config.

Deprecated since version 0.3.0: hydra_run will be removed in hydra-zen 1.0.0; it is replaced by hydra_zen.launch().

Parameters
configUnion[DataClass, DictConfig, Mapping]

A configuration as a dataclass, configuration object, or a dictionary.

task_functionCallable[[DictConfig], Any]

The function Hydra will execute with the given configuration.

overridesOptional[List[str]] (default: None)

If provided, overrides default configurations, see [1] and [2].

config_dirOptional[Union[str, Path]] (default: None)

Add configuration directories if needed.

config_namestr (default: “hydra_run”)

Name of the stored configuration in Hydra’s ConfigStore API.

job_namestr (default: “hydra_run”)
with_log_configurationbool (default: True)

Flag to configure logging subsystem from the loaded config

Returns
resultJobReturn
